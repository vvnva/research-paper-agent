# Governance

### Risk Register

| Риск | Вероятность | Влияние | Детект | Защита | Остаточный риск |
|-----|-----|-----|-----|-----|-----|
| Prompt injection в тексте статьи | Средняя | Среднее | Анализ входных данных | Строгая структура prompt, экранирование входных данных | Низкий |
| Галлюцинации LLM | Средняя | Среднее | Проверка источников | Ограничение генерации только на основе abstract | Средний |
| Нерелевантные результаты поиска | Высокая | Низкое | Анализ пользовательского запроса | Query rewriting и reranking | Низкий |
| Ошибка внешнего API | Средняя | Среднее | Проверка статуса API | Retry механизм и fallback | Низкий |
| Превышение лимитов API | Низкая | Среднее | Мониторинг количества запросов | Ограничение частоты запросов | Низкий |

---

## Политика логирования

Система сохраняет минимальный набор технических логов:

- время запроса
- статус выполнения запроса
- ошибки API
- технические сообщения системы

Не сохраняются:

- персональные данные пользователя
- содержимое сообщений пользователя
- результаты поиска

Логи используются только для отладки и мониторинга системы.

---

## Работа с персональными данными

PoC не собирает и не хранит персональные данные пользователей

Telegram ID пользователя может использоваться только для управления сессией и не сохраняется в долгосрочном хранилище

Система не требует регистрации пользователя

---

## Защита от Prompt Injection

Для защиты от prompt injection используются следующие меры:

- внешний текст (abstract статей) обрабатывается как данные
- пользовательский ввод не используется в system prompt
- применяется строгая структура prompt
- ввод пользователя ограничивается тематическими запросами

---

## Подтверждение действий

Система не выполняет опасных действий от имени пользователя.
Все действия ограничиваются:

- поиском научных статей
- анализом метаданных
- генерацией кратких описаний
