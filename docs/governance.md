## Политика логирования

Система сохраняет минимальный набор технических логов:

- время запроса
- статус выполнения запроса
- ошибки API
- технические сообщения системы

Не сохраняются:

- персональные данные пользователя
- содержимое сообщений пользователя
- результаты поиска

Логи используются только для отладки и мониторинга системы.

---

## Политика хранения данных

Система использует ограниченное session state

- Данные сессии (query, результаты, фильтры) хранятся временно  
- Время хранения ограничено TTL (по умолчанию не более 1 часа)  
- После истечения TTL данные автоматически удаляются  
- Долгосрочное хранение пользовательских данных отсутствует 

---

## Работа с персональными данными

PoC не собирает и не хранит персональные данные пользователей

Telegram ID пользователя может использоваться только для управления сессией и не сохраняется в долгосрочном хранилище

Система не требует регистрации пользователя

Система не обрабатывает и не хранит персональные данные пользователей, поэтому проведение оценки соответствия требованиям GDPR для данного PoC не требуется

---

## Защита от Prompt Injection

Для защиты от prompt injection используются следующие меры:

- внешний текст (abstract статей) обрабатывается как данные
- пользовательский ввод не используется в system prompt
- применяется строгая структура prompt
- ввод пользователя ограничивается тематическими запросами

---

## Подтверждение действий

Система не выполняет деструктивных или чувствительных действий от имени пользователя

Все действия ограничиваются:

- поиском научных статей
- анализом метаданных
- генерацией кратких описаний

Дополнительное подтверждение действий от пользователя не требуется, так как:

- отсутствуют операции изменения данных  
- отсутствуют внешние side-effect действия  
- отсутствует доступ к пользовательским аккаунтам или ресурсам  

Таким образом, риск ошибочного или нежелательного действия со стороны системы минимален
